Шаги создания сценария по подготовке данных для Загрузки - тестирование "Заказ Клиента"

1. Запустил обработку по "Подготовке и загрузке данных" (VA)
2. Указал, что мне нужны для загрузки:
 - Все константы
 - Документ "Заказ Клиента"
 - Регистр накопления "Заказ Клиентов"
3. Нажал на кнопку "Выбрать зависимые элементы". 
 - предварительно на закладке отбор по ссылка указал "Включать восходящие зависимости" + выбрал 3 (три) документа для загрузки = с разными условиями по работе с Клиентами.

5. Создал тест Фичи.
6. Перенес текст Фичи в новый сценарий.
 - Изначально делал с анегл. языком для Фичи - но оказалось, что в таком подходе есть и плюс и минус:
 -- ПЛЮС: все данные были корректно представлены в текстовом документе - те. разбивка текста по блокам была структурирована и читаема
 -- МИНУС: В некоторых ШАГАХ количество параметров в русском и Английском варианте было разным (вероятно ошибка разработчиков)
 - ПО этой причине перегенерировал тест ФИЧИ на русском языке и сохранил файл.
7. Во внешнем текстовом редакторе произвел следующие изменения:
 - Убрал лишние строки в тексте сценария
 - Заменил конкретные наименования на анонимные = более "говорящие" для процесса тестирования
(это был самый трудоемкий процесс с упором на внимательность...)
 - Разделил на несколько сценариев, выделив предварительные - создание справочников и ПланаВидовХарактеристик - во внешние сценарии
 - Подключил внешние сценарии к основному:
 -- Перенес внешние сценарии в новый каталог "Библиотеки" внешних сйенариев
 -- отметил тегами "@ExportScenarios" и "@IgnoreOnCIMainBuild" экспортные сценарии
 -- Переименовал по иерархии исполнения файлы внешних сценариев (с префиксом "001_" и "002_")
 -- Подлючил внешние сценарии к основному через команду "И"
8. Проверил, текст внешних сценариев "подтягивается" к тексту основного сценария выполнения загрузки данных.
9. Проверил загрузку данных:
 - Сохдал новую базу
