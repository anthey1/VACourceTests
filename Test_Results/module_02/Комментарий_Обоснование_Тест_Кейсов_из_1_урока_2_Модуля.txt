Наиболее часто используемые техники тестирования (тест-дизайна)

1. Граничные Значения
2. Классы эквивалентности
3. Граница решений
4. Предугадывание ошибки
5. Причина / Следствие

Задание для тестирования:
При закупке 1000 (вкл) до 2000 = предоставляемая скидка 5%
Свыше 2000 (вкл) = предоставляемая скидка 10%

-----------
Предлагаемое решение = варианты тестов:

1. Проверка Значения Скидки на стоиомсть "0" (в случае, если разрешена реализация в качестве "подарка" или ошибка оператора)
(обоснование = возможна ошибка в математических операциях, да и какая скидка вообще при стоимости "0")

2. проверка на значение Скидки "0" при условии "стоимость > 0 И стоимость <= 1000
(Обоснование - требование математического условия в задаче)

3. проверка на значение Скидки "5" при условии "стоимость > 1000 И стоимость <= 2000
(Обоснование - требование математического условия в задаче)

4. проверка на значение Скидки "10" при условии "стоимость < 2000
(Обоснование - требование математического условия в задаче)

5. проверка на значение Скидки > "10"
(Обоснование - Определение некорректных условий, когда скидка может рассчитываться в заданных диапазонах стоимсти по неверным правилам)
